# v0.7.4

## Summary

- Frontend scroll profiling now captures active scrolling frames, main RunLoop active time, long-frame attribution, main-thread coverage, accessibility tree snapshots, and XCTest accessibility query timing.
- History rows now prewarm presentation-only derived values for Markdown export and file preview metadata, so row rendering avoids repeated parsing while keeping existing copy, preview, export, and selection behavior unchanged.
- Scroll-time work is reduced by suppressing hover/animation churn while scrolling and by deferring thumbnail commits until scrolling settles.
- Thumbnail loading now dedupes in-flight decodes and uses bounded decode concurrency with separate queue, ImageIO decode, main commit, and total-load metrics.

## Key Changes

- Added `HistoryItemPresentationCache` for row-level derived presentation values that are safe to precompute off the main thread.
- Wired display prewarm through `HistoryViewModel` item load/search/update paths and consumed cached values from `HistoryItemView`.
- Expanded row equality inputs to cover visible content-affecting fields while recording `swiftui.row_equatable_ms`.
- Added `swiftui.row_body_ms`, `row.display_model_ms`, `row.file_preview_ms`, thumbnail decode/queue/commit metrics, and long-frame attribution summaries to frontend profiles.
- Updated `make perf-frontend-profile-full` and `make perf-unified-table` so release evidence includes frontend scroll stability and backend search/warm-load metrics in one report.
