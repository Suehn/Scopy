# v0.7.9

## Summary

This release restores the earlier clipboard panel behavior where search-field focus and row selection are independent. Focusing the search field no longer clears the selected row, and hover can still update the selected row while the field remains focused.

It also includes the post-`v0.7.8` test-host fix that keeps the AppDelegate Codex `Control+V` shortcut assertion out of the TSan host bundle, matching the TSan target's intentional `AppDelegate.swift` exclusion.

## Key Changes

### Search Focus And Selection

- Removed the `HistoryViewModel.clearSelectionForSearchFocus()` path.
- Removed the `ContentView` focus-change hook that cleared selection when the search field gained focus.
- Removed the `HistoryListView` hover guard that blocked row selection while the search field was focused.
- Updated product, development, and Trellis state-management docs so the executable contract is explicit: search typing focus and visible row selection are independent.

### Test Coverage

- Replaced the hover suppression UI expectation with coverage that requires hover selection to work while the search field is focused.
- Removed obsolete unit coverage for search-focus-driven selection clearing.
- Excluded `AppDelegateTests` from `SCOPY_TSAN_TESTS`, because the TSan host excludes `AppDelegate.swift` and cannot reference the app-target shortcut constant.

## Performance Evidence

No dedicated release profile was added for `v0.7.9`; `profile_doc` is `null` in release metadata. The release changes focus/selection state handling and test-host compilation only, with no search, storage, thumbnail, or scrolling hot-path changes.
