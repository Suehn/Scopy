# v0.8.7

## Summary

This patch corrects the Markdown table parity model introduced in `v0.8.6`. Ordinary Markdown pipe tables now follow the captured WACZ root Markdown table path, while the separate ChatGPT TableContainer component model remains documented as a distinct renderer surface.

## Key Changes

### Markdown Preview

- Measures ordinary Markdown pipe-table columns from rendered header and body text and assigns the official `data-col-size` attribute plus Scopy's diagnostic mirror.
- Uses the WACZ pipe-table size buckets only: `>160 -> xl`, `>100 -> lg`, `>40 -> md`, otherwise `sm`.
- Removes the component-only `xs` bucket from ordinary Markdown pipe tables.
- Sizes pipe-table columns against the root thread content max width instead of the TableContainer component `640px` / `768px` baseline.
- Restores the root Markdown table rhythm: body text metrics, zero base inline padding, 24px end padding between non-last cells, 8px header block padding, 10px body block padding, and extra bottom space on the last row.
- Preserves Scopy's local horizontal scroll wrapper while keeping the Markdown table model itself aligned with WACZ.

### Markdown Export

- Keeps PNG export on the same preview-equivalent Markdown table DOM and CSS before applying the export-only bitmap fitting pass.
- Updates export diagnostics and fallback wrapping so table measurements target the local `container > wrapper > table` structure.

### Documentation And Tooling

- Updates the WACZ Markdown style contract to separate root Markdown pipe-table behavior from the TableContainer component model.
- Updates the WACZ analyzer summary so its reported table thresholds match the extracted Markdown/rehype table path.
- Extends the renderer regression test to assert the absence of the old `xs`, `length > 4`, and `640px` / `768px` component baseline assumptions for ordinary Markdown pipe tables.
