# v0.7.8

## Summary

This release fixes the clipboard panel action contracts that surfaced after `v0.7.7`: pinned rows no longer reduce the first recent page, image AirDrop is available for both file-backed and inline images, and the Codex optimized paste path sends the expected `Control+V` keystroke after copying.

The key implementation boundary is now service-owned file action resolution. Views decide whether to show an action from DTO-visible capability hints, while `ClipboardServiceProtocol.fileURLs(itemID:)` resolves validated file URLs or prepares a temporary PNG for stored images that do not have an original file on disk. Open Containing Folder remains stricter and only uses real file-backed URLs.

## Key Changes

### History Loading

- `HistoryViewModel.load()` now fetches pinned items with `fetchPinned()` and recent unpinned rows with `fetchRecentUnpinned(limit:offset:)`.
- Pinned rows remain visually grouped without consuming the `initialPageSize = 50` recent-page quota.
- `loadMore()` fetches 500 additional recent unpinned rows at a time and offsets from the loaded unpinned count.

### Context Menu Actions

- All image rows expose Send via AirDrop. File-backed images share their original file URL, and inline/stored images generate a temporary PNG under the system temporary directory for sharing.
- File rows expose Send via AirDrop and Open Containing Folder only when resolvable local non-directory files exist.
- Open Containing Folder ignores temporary image-share PNGs so the action only reveals user-meaningful source files.
- Non-file text, RTF, HTML, and other rows keep the copy/pin/delete-only file-action surface.

### Codex And Panel UX

- Paste-optimized for Codex now copies the optimized representation, closes the panel, and posts `Control+V`.
- Search focus clears row selection and suppresses immediate hover reselection while the search field remains focused.
- Reopening the panel after the stale-search interval clears old search text; quick reopen keeps the active query.

## Performance Evidence

No dedicated release profile was added for `v0.7.8`; `profile_doc` is `null` in release metadata. The pagination change reduces initial unpinned work when pinned rows exist, and the AirDrop PNG path is only entered from an explicit context-menu action.
