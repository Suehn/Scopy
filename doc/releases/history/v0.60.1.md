# v0.60.1

## 📌 一页纸总结（What / Why / Result）

### What

- 后端清理路径性能收敛：
  - count-plan 外部空间释放估算改为数据库聚合，去掉逐文件 `stat` 扫描。
  - external-plan 支持排除 count-plan 已选 ID，减少重复规划与二次清理。
- 前端真实性能基线完善：
  - 新增/固化 `make perf-frontend-profile` 与 `make perf-unified-table`。
  - 将前端 profile 拆分为 smoke/standard/full 三档，默认开发跑 smoke，发布前跑 full。
  - UI profile 滚动驱动改为窗口坐标拖拽，避免 AX 句柄失效导致的偶发失败。
- release 性能门禁加固：
  - `test-snapshot-perf-release` 增加 p95 数值解析校验，防止解析异常假绿。

### Why

- 之前 external cleanup 在 10k 外部项场景出现过边界超时（~1918ms），且组合清理存在重复选集与额外流程开销。
- 前端需要“真实 snapshot DB + baseline/current 同口径”对比，避免玩具数据误导。
- release 性能门禁必须是“失败可见、不可假绿”。

### Result

- External cleanup 10k：`1918.01ms`（失败样本）→ `1019.45ms`（通过样本）。
- Release bench（snapshot DB）：
  - `cmd` p95 = `0.180ms`（target 50ms）
  - `cm` p95 = `5.833ms`（target 20ms）
- 前端真实场景 profile 与前后端统一同表已产出：
  - `logs/perf-frontend-profile-2026-02-28-final/`
  - `logs/perf-unified-2026-02-28-front-back-final.md`

## 🏗️ 实现路线

1. 收敛 cleanup 热点：把释放字节估算改到 DB 侧，并修复组合计划重叠选集。

4. 修复 UI profile 稳定性：滚动驱动从 AX 列表元素 swipe 改为窗口坐标拖拽。
5. 全量/关键门禁验证并更新实现文档、评审文档、操作指南。
