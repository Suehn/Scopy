# v0.8.0

## Summary

This release completes the unified Markdown renderer hardening work for hover preview and Markdown export. Scopy now requires the new renderer chain to finish successfully before showing or caching rendered Markdown, rather than treating raw Markdown fallback content as a successful preview.

The release also fixes trusted plugin deep links inside rendered Markdown. Links such as `plugin://computer-use@openai-bundled` remain clickable after sanitization, while local filesystem links are still stripped from rendered HTML.

## Key Changes

### Unified Markdown Renderer Contract

- The bundled unified renderer sanitizer now extends the default schema with the trusted `plugin` protocol for `href` attributes.
- Unsafe local file links such as `file:///Users/...` remain blocked in rendered Markdown.
- The Swift preview document exposes explicit `renderSucceeded` and `renderErrorReason` state from the WebView readiness bridge.
- The legacy raw `<pre>` fallback path is removed from the unified preview document so render failures surface as failures.

### Hover Preview Liveness

- Hover preview state now tracks whether the Markdown render really succeeded.
- Rendered preview size and cache updates are only accepted from the currently live hover request.
- Stale hover work is dropped after rendering rather than blocking rendering before the final liveness check.
- Popover transition keeps the current Markdown render alive long enough to avoid tearing down the active preview.

### Markdown Export

- Markdown PNG export now fails at prepare-layout time when the unified renderer reports failure.
- Export debug output includes the renderer success/error state for easier postmortem diagnosis.
- Markdown preview and export continue sharing the same bundled unified renderer artifact and checksum.

### Test Coverage

- Added JavaScript renderer coverage for plugin deep links, unsafe file links, and long Chinese reference-style Markdown.
- Added Swift unit coverage for hover-preview liveness, Markdown detection/source classification, renderer contract strings, and render-before-final-liveness behavior.
- Added UI coverage that waits for the Markdown preview popover to report rendered status before asserting file and long-text Markdown previews.

## Performance Evidence

No dedicated release profile was added for `v0.8.0`; `profile_doc` is `null` in release metadata. The release changes Markdown rendering, hover-preview liveness, and export failure handling only, with no search, storage, thumbnail, or scrolling hot-path changes.
