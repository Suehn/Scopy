# v0.8.3

## Summary

This patch aligns Scopy's shared preview/export Markdown HTML theme with the rendering model extracted from the local ChatGPT WACZ archives. The theme now covers typography, heading rhythm, paragraphs, inline code, fenced code cards, code language labels, syntax colors, blockquotes, lists, task lists, links, footnotes, horizontal rules, and tables from the same base style.

The change keeps preview/export layout semantics shared: normal text lays out at the available ChatGPT content width, wide tables scroll across the responsive render surface while staying anchored at the message-column inset, and PNG export scales the already-laid-out table surface instead of substituting a second export-only table style.

## Key Changes

### Markdown Theme

- Replaces Scopy's previous card-like Markdown surface with WACZ-derived ChatGPT typography, spacing, colors, code-card treatment, syntax colors, quote bar, list metrics, link decoration, footnote treatment, and horizontal-rule spacing.
- Keeps heading-contained code spans on the same ChatGPT gray inline-code pill path as paragraph/list/table/quote inline code, with relative heading-aware sizing.
- Promotes explicit parenthesized source-reference links such as `([AP News][1])` into ChatGPT-style 18px citation pills in both preview and PNG export, collapses grouped source references such as `([AP News][1], [Reuters][2])` to the first source plus a `+1` suffix, while preserving ordinary parenthesized links such as `([guide][1])` as normal dotted-underlined links.
- Shares ATX heading normalization across legacy and unified rendering so malformed heading source such as `#标题` is normalized before Markdown parsing instead of drifting into paragraph styling on one renderer path.
- Normalizes unescaped pipe characters inside table-row one- or two-backtick inline code spans before GFM parsing, keeping examples such as `` `| A | B |` `` in one table cell across the Swift preview/export path and the bundled unified renderer, while preserving three-backtick fence-marker examples such as ```` ```python ```` as literal table-cell text.
- Bumps Markdown render context policy/cache namespaces after the parser and bundled renderer semantics changed, so stale hover-preview HTML generated by the older table parser cannot keep showing exploded rows.
- Re-anchors Scopy's default ChatGPT width model to the currently measured `40rem`/`640px` Markdown root plus 24px side padding (`688px` render surface). The larger `48rem` ChatGPT branch remains documented as a container-breakpoint behavior, not the default Scopy hover/export width.
- Records the source-derived WACZ style contract in `doc/current/markdown-chatgpt-wacz-style-contract.md` so future Markdown changes start from CSS/DOM evidence rather than screenshots.
- Syncs the modern AssistantMessage spacing/color contract for headings, paragraphs, lists, blockquotes, inline code pills, horizontal rules, and table borders, including the source 4px blockquote vertical padding and full-height quote bar.
- Reworks task-list rendering to match the AssistantMessage source contract: no list bullets, baseline flex rows, an 8px marker/text gap, and one CSS-painted visual marker path for both generated checkbox inputs and raw `[x]` text markers in WKWebView.
- Aligns light-theme `hljs` syntax token colors with the WACZ CodeBlock semantic palette rather than Scopy's earlier approximate palette, and keeps inline-code contrast on the captured alpha-04 background plus alpha-08 inset stroke.
- Keeps `display: block`, `overflow-x: auto`, source-derived table breakout margins, and `table-layout: auto` in Scopy's table CSS so preview tables scroll without widening the preview frame, while table cell wrapping follows the WACZ-style natural line breaking and cells keep the source 8px header block padding, 10px body block padding, 8px non-first-column start padding, 24px non-last-column end padding, 40px last-column end padding, 16px header line height, and 24px last-row bottom padding.

### Scaling Boundary

- Changes `MarkdownExportService` wide-table scaling to use the same unscaled content-box target as preview. Global export scaling no longer widens `#content` by `1 / scale`, preserving normal paragraph line breaks and table measurements before bitmap scaling.
- Keeps global export scale compatible with the injected export stylesheet by overriding `#content` width at the same CSS priority, while preserving the Scopy Markdown `#content-scale-shell` layout width before bitmap scaling.
- Extends focused Markdown renderer unit tests to assert the WACZ-aligned non-table style contract, footnote treatment, table visual contract, and existing overflow contract.

### Build Tooling

- Keeps resource-staging run script phases correctness-first, but avoids repeated full MarkdownPreview asset copies and unnecessary pngquant recopy work during no-op builds.
