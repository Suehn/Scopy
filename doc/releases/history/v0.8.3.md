# v0.8.3

## Summary

This patch aligns Scopy's shared preview/export Markdown HTML theme with the rendering model extracted from the local ChatGPT WACZ archives. The theme now covers typography, heading rhythm, paragraphs, inline code, fenced code cards, code language labels, syntax colors, blockquotes, lists, task lists, links, footnotes, horizontal rules, and tables from the same base style.

The change keeps preview/export layout semantics shared: normal text lays out at the ChatGPT content width, wide tables scroll inside that preview width, and PNG export scales the already-laid-out table surface instead of substituting a second export-only table style.

## Key Changes

### Markdown Theme

- Replaces Scopy's previous card-like Markdown surface with WACZ-derived ChatGPT typography, spacing, colors, code-card treatment, syntax colors, quote bar, list metrics, link decoration, footnote treatment, and horizontal-rule spacing.
- Keeps heading-contained code spans visually aligned with the heading text itself, while paragraph/list/table/quote inline code retains the ChatGPT gray pill treatment.
- Syncs the modern AssistantMessage spacing/color contract for headings, paragraphs, lists, blockquotes, inline code pills, horizontal rules, and table borders, including the source 4px blockquote vertical padding and full-height quote bar.
- Reworks task-list rendering to match the AssistantMessage source contract: no list bullets, baseline flex rows, an 8px marker/text gap, and one CSS-painted visual marker path for both generated checkbox inputs and raw `[x]` text markers in WKWebView.
- Aligns light-theme `hljs` syntax token colors with the WACZ root stylesheet rather than Scopy's earlier approximate palette.
- Keeps `display: block`, `overflow-x: auto`, `width: 100%`, and `table-layout: auto` in Scopy's table CSS so preview tables scroll without widening the preview frame, while table cell wrapping follows the WACZ-style natural line breaking and cells keep the source 8px inline padding.

### Scaling Boundary

- Changes `MarkdownExportService` wide-table scaling to use the same unscaled content-box target as preview. Global export scaling no longer widens `#content` by `1 / scale`, preserving normal paragraph line breaks and table measurements before bitmap scaling.
- Keeps global export scale compatible with the injected export stylesheet by overriding `#content` width at the same CSS priority, while preserving the Scopy Markdown `#content-scale-shell` layout width before bitmap scaling.
- Extends focused Markdown renderer unit tests to assert the WACZ-aligned non-table style contract, footnote treatment, table visual contract, and existing overflow contract.

### Build Tooling

- Keeps resource-staging run script phases correctness-first, but avoids repeated full MarkdownPreview asset copies and unnecessary pngquant recopy work during no-op builds.
