# v0.8.5

## Summary

This patch preserves file-backed image paste behavior for apps that can accept file transfers, while keeping PNG fallback data for image-only paste targets. It also releases the README and brand-visual refresh that landed after `v0.8.4`.

## Key Changes

### Clipboard

- Replays managed file-backed image history items with a file URL representation plus PNG fallback data.
- Keeps inline or non-file-backed image history items on the existing PNG replay path.
- Keeps the Codex-optimized image replay path unchanged so its explicit TIFF compatibility behavior remains isolated.
- Validates Scopy-managed image `storageRef` paths before exposing them as pasteboard file URLs.

### Documentation And Visuals

- Refreshes README product messaging around real Scopy `FloatingPanel` screenshots, visual previews, quick install links, and a denser feature matrix.
- Adds app icon assets and preview imagery used by the README and app bundle metadata.
