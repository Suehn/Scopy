# v0.8.1

## Summary

This patch release restores syntax-highlighted code blocks on the unified Markdown renderer path while keeping the v0.8.0 hardening contract intact. Scopy now emits `hljs` token classes from the unified bundle itself, so the existing preview/export code theme applies without loading the legacy markdown-it highlighter runtime.

The fix intentionally avoids language auto-detection. Explicit language fences are highlighted; unlabeled fences and explicit plain-text fences remain literal to prevent raw HTML examples, file paths, shell variables, and prose-like snippets from being rewritten by a guessed grammar.

## Key Changes

### Unified Markdown Code Highlighting

- Added `rehype-highlight` after sanitization in the unified renderer pipeline.
- Reused Scopy's existing `.hljs-*` preview/export theme instead of introducing a second theme surface.
- Added common language aliases for user-facing fence labels such as `sh`, `shell`, `zsh`, `js`, `jsx`, `ts`, `tsx`, `md`, `py`, `yml`, and Objective-C labels.
- Marked `text`, `txt`, `plain`, and `plaintext` as plain text fences.

### Regression Boundaries

- Kept highlighter auto-detection disabled to avoid rewriting unlabeled code fences.
- Preserved the v0.8.0 unified-renderer success/failure readiness contract.
- Kept unified preview documents free of `markdown-it`, `highlight.min.js`, and `highlight-github.min.css` runtime fallback scripts.

### Test Coverage

- Added JavaScript renderer tests for highlighted fenced code blocks and explicit plain-text fences.
- Updated corpus coverage so shell fences require `hljs` token output while preserving local path literals.
- Added Swift shell assertions that unified preview documents carry the highlight theme but do not load the old highlight runtime.

## Performance Evidence

No dedicated release profile was added for `v0.8.1`; `profile_doc` is `null` in release metadata. The release changes the bundled Markdown renderer dependency graph and generated renderer asset only, with no search, storage, thumbnail, or scrolling hot-path changes.
