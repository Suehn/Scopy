# v0.7.2

## Summary

- Markdown preview/export now preserves CJK punctuation-adjacent asterisk emphasis without leaking internal renderer sentinels.
- History loading is guarded against stale async writeback when a newer search or filter state wins the UI.
- Thumbnail settings resets now keep disk cache clearing, in-memory thumbnail indexing, and DTO thumbnail regeneration in sync.
- Release version resolution now follows the nearest valid release tag instead of historical version-sort order.

## Key Changes

- Added `MarkdownCJKEmphasisNormalizer` and wired it into `MarkdownHTMLRenderer` before safe HTML restoration and local `markdown-it` rendering.
- Hardened `HistoryViewModel.load()` with search-version, cancellation, and unfiltered-list guards after awaited work.
- Hardened thumbnail cache reset by invalidating `ClipboardService`'s thumbnail index around `StorageService.clearThumbnailCache()` and verifying indexed paths still exist.
- Made `StorageService.clearThumbnailCache()` await thumbnail directory removal and recreation.
- Updated `scripts/version.sh` and `scripts/build-release.sh` so local and release builds share one valid-tag resolver.
