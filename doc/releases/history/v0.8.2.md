# v0.8.2

## Summary

This patch release fixes Markdown documents that mix normal Markdown with safe HTML islands such as `<details>`, `<kbd>`, `<mark>`, `<sub>`, and `<sup>`. These sources now keep the v0.8 unified renderer path when they have strong Markdown structure outside the safe HTML islands, so hover preview and PNG export render the document instead of falling back to raw Markdown text.

The release also hardens the legacy renderer path and the routing boundary. Rich HTML containers such as `<div>`, `<span>`, `<table>`, `<pre>`, `<code>`, and `<blockquote>` still stay on the conservative legacy path even when their inner text contains Markdown-like markers. Safe-HTML placeholder replacement is now lazy and only renders placeholders that are actually present in the current HTML fragment, avoiding details recursion and unrelated placeholder side effects.

## Key Changes

### Safe HTML Markdown Routing

- Detects authored Markdown with safe HTML islands before classifying a source as rich HTML, but requires Markdown signal outside the safe HTML islands before taking the unified renderer path.
- Leaves pure HTML-like clipboard content and non-safe raw HTML containers on the conservative legacy markdown-it path.
- Adds corpus coverage for safe-HTML Markdown so future renderer cutovers cannot silently route it back to legacy.

### Legacy Details Stability

- Makes safe-HTML replacement lazy by checking placeholder presence before rendering replacement HTML.
- Uses replacement callbacks for paragraph placeholder replacement so details rendering only happens after a real match.
- Skips absent safe-HTML placeholders during nested replacement so unrelated tokens cannot trigger nested rendering work.
- Keeps forced legacy behavior available for rollback/debugging without reintroducing the details recursion failure.

### Export Regression Coverage

- Adds a UI export fixture covering headings, quotes, lists, inline safe HTML, `<details>`, highlighted code, tables, footnotes, and KaTeX.
- Registers that fixture in the Xcode project so clean checkout UI tests package the same resource.
- Requires the exported PNG to have rendered content dimensions, catching the previous raw Markdown fallback export.

## Performance Evidence

No dedicated release profile was added for `v0.8.2`; `profile_doc` is `null` in release metadata. The change is limited to Markdown renderer routing, the generated HTML shell, and renderer/export tests. It does not touch search, storage, thumbnail, pagination, or scrolling hot paths.
