# v0.8.2

## Summary

This patch release fixes Markdown documents that mix normal Markdown with safe HTML islands such as `<details>`, `<kbd>`, `<mark>`, `<sub>`, and `<sup>`. These sources now keep the v0.8 unified renderer path when they have strong Markdown structure, so hover preview and PNG export render the document instead of falling back to raw Markdown text.

The release also hardens the legacy renderer path. The old safe-HTML `<details>` replacement logic eagerly rendered every placeholder even inside nested fragments, which could recursively render the outer `<details>` placeholder until JavaScript hit a maximum call stack error. The replacement is now lazy and only renders placeholders that are actually present in the current HTML fragment.

## Key Changes

### Safe HTML Markdown Routing

- Detects authored Markdown before classifying a source as rich HTML, so Markdown documents with safe inline HTML and details blocks use the unified renderer.
- Leaves pure HTML-like clipboard content on the conservative legacy markdown-it path.
- Adds corpus coverage for safe-HTML Markdown so future renderer cutovers cannot silently route it back to legacy.

### Legacy Details Stability

- Makes legacy safe-HTML replacement lazy by checking placeholder presence before rendering replacement HTML.
- Uses replacement callbacks for paragraph placeholder replacement so details rendering only happens after a real match.
- Keeps forced legacy behavior available for rollback/debugging without reintroducing the details recursion failure.

### Export Regression Coverage

- Adds a UI export fixture covering headings, quotes, lists, inline safe HTML, `<details>`, highlighted code, tables, footnotes, and KaTeX.
- Requires the exported PNG to have rendered content dimensions, catching the previous raw Markdown fallback export.

## Performance Evidence

No dedicated release profile was added for `v0.8.2`; `profile_doc` is `null` in release metadata. The change is limited to Markdown renderer routing, the generated HTML shell, and renderer/export tests. It does not touch search, storage, thumbnail, pagination, or scrolling hot paths.
