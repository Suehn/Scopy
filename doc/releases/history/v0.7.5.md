# v0.7.5

## Summary

- History rows now reuse presentation-only display text and file preview summaries, reducing repeated string/path scans in row construction, thumbnail eligibility checks, hover preview, and Markdown export capability checks.
- Full-index fuzzy search now caches a bounded first-page top-K prefix, so the immediate next page can reuse work already paid for by the first full fuzzy scan.
- Settings updates now apply cleanup, thumbnail cache invalidation, and history reloads only when the changed setting actually affects those paths.

## Key Changes

- Added `FilePreviewSupport.previewSummary` and reused it across `ClipboardService`, `HistoryItemPresentationCache`, and file preview rendering while preserving the `requireExists` boundary between display-time metadata and thumbnail generation.
- Added paired display-text computation in `ClipboardItemDisplayText`, so title and metadata share one derivation and file entries share one path summary.
- Reused cached Markdown-export capability in hover text previews and offloaded uncached detection to utility-priority work with cancellation checks before UI state updates.
- Extended `SearchEngineImpl`'s fuzzy sorted-match cache to first-page full fuzzy scans behind `SCOPY_PERF_FUZZY_FIRST_PAGE_CACHE`, with perf counters for cache hits and stored top-K entries.
- Added `SettingsPatch` helper properties and used them to avoid no-op storage cleanup, thumbnail cache clears, and history reloads while keeping hotkey reapplication behavior compatible.
