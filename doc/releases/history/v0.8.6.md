# v0.8.6

## Summary

This patch completes the WACZ TableContainer parity pass for Markdown tables. Preview now follows the captured ChatGPT table component layout directly, while PNG export starts from that same preview-equivalent layout and applies only the export fitting layer afterward.

## Key Changes

### Markdown Preview

- Applies the WACZ TableContainer column-sizing model to every Markdown table instead of only inferred-wide tables, so normal two-column summary tables and wider comparison tables share the same `xs/sm/md/lg/xl` sizing contract.
- Keeps table column widths based on the captured component baseline of `640px`, switching to `768px` at `min-width: 1024px`, instead of binding column widths to Scopy's current message/content width.
- Restores the missing inner fit-content table wrapper from the WACZ component path, so horizontal overflow is governed by the same `container > wrapper > table` structure as the official renderer.
- Replaces the older native Markdown table padding with the WACZ component cell model: base 8px horizontal padding, zero first-cell start padding, zero last-cell end padding, 8px header-row block padding, and 10px body-row block padding.
- Keeps browser-zoomed previews on the layout-only table path, allowing horizontal overflow when the official renderer would overflow instead of forcing premature cell wrapping.

### Markdown Export

- Moves table transform fitting behind an export-only `__scopyScaleChatGPTTablesForExport` hook.
- Keeps preview and export table layout shared before export scales oversized table surfaces to fit the bitmap target.
