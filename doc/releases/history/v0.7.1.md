# v0.7.1

## 📌 一页纸总结（What / Why / Result）

### What

- Markdown/Footnotes：
  - 修复 `MathNormalizer` 对 markdown footnote 语法的误判，保证 `[^1]` / `[^1]: ...` 继续进入 `markdown-it-footnote`。
  - 脚注引用统一为蓝色上标数字，底部 footnote caption 直接输出裸数字而不是 `[1]` 这类括号文本。
- Markdown/Export：
  - 代码高亮加入内联 token 颜色兜底，导出 PNG 不再只依赖外部主题 CSS 的加载时序。
  - 导出链路的 render-ready 判断兼容 HTML-only harness，避免测高阶段因为等待不存在的 hook 而退化。
- Test/Release：
  - 补上 footnote 保护与 renderer/export 回归，锁住 release build、preview 和 PNG export 的一致性。

### Why

- `v0.7.0` 已把 Markdown 能力面扩到 `CommonMark + GFM + footnotes + math`，但后续 release-mode 实测暴露出两个剩余缺口：脚注引用仍可能退化成普通 `[1]` 文本，代码高亮在导出图里也可能只剩灰度明暗差。
- 根因不是单一 CSS 主题问题。脚注问题来自 `MathNormalizer` 过早触碰 `[^1]` 语法；高亮问题来自 export 对主题加载时序的依赖过强，以及 render-ready 检查对非 markdown harness 过严。

### Result

- 预览、release build 和 PNG export 现在对脚注语义保持一致：正文里是蓝色上标数字，底部是正常 footnote section。
- 代码块在导出图中继续保留彩色 token，高亮不再因为外部 CSS 时序波动而退化成纯灰度块。
- 本次改动是 `v0.7.0` 的语义补丁，不改变 Markdown 栈、不改变 preview/export 的整体契约，只收口已经确认的偏差。

---

## 🏗️ 实现路线

1. 在 `MathNormalizer` 里为 markdown footnote 语法增加豁免，阻止 loose-math 预处理误吞脚注标记。
2. 在 `MarkdownHTMLRenderer` 中同时修正 footnote 样式和 caption 生成，把脚注引用从普通 bracket text 收口成蓝色上标数字。
3. 为 highlight token 增加 renderer 内联颜色兜底，并放宽 export render-ready 兼容条件，避免 release/export 路径再出现“预览有色、导出发灰”。
4. 补充 unit/UI 回归并保持现有长图导出与标准 markdown case 测试继续通过。
