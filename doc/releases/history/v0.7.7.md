# v0.7.7

## Summary

This release documents the architecture quality pass after `v0.7.6`. It keeps the visible clipboard, search, preview, export, settings, and selection surface stable while narrowing internal ownership boundaries for hover preview, storage cleanup, exact-search intent, Trellis archive context paths, and quality gate evidence.

The biggest product-code change is `HistoryHoverPreviewPipeline`: row views now submit typed preview requests for text, Markdown file, image, and file-preview flows while the pipeline owns planning, cache-hit/cache-miss events, suppression checks, and bounded detached preview work. Storage cleanup now routes delete plans through one executor, and exact search now shares trimmed-query normalization between planning and execution.

## Key Changes

### Frontend Architecture

- Added `HistoryHoverPreviewPipeline` for hover-preview planning across text, Markdown file, image, and file preview cases.
- Kept `HistoryItemView` focused on rendering, state application, popover presentation, and interaction handoff instead of raw preview planning.
- Added hover-preview profile smoke support through `scripts/perf-frontend-profile.sh --include-hover`, with required Markdown render and image decode buckets.

### Search And Storage

- Exact search planning and execution now share `SearchPlanner.normalizedExactQuery(_:)`, so whitespace-only and whitespace-padded exact queries use the same trimmed intent for recent-only versus complete-history behavior.
- Storage cleanup by count, age, size, image-only, external-storage, and composite cleanup now executes repository delete plans through the shared `StorageService.applyDeletePlan` adapter.
- `deleteAllExceptPinned()` remains separate because it needs a tighter repository transaction boundary for all unpinned rows and storage refs.

### Tooling And Trellis Stability

- Added `scripts/quality/record-gate-result.py` and `make quality-manifest-self-test` to record quality gate results into JSON and Markdown manifests without replacing the existing build or test commands.
- Trellis archive now rewrites task-local `implement.jsonl` and `check.jsonl` file entries when tasks move under `.trellis/tasks/archive/YYYY-MM/`.
- Added regression coverage for archived task context path rewriting so archived PRD and research links remain valid.
