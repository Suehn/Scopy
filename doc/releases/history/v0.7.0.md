# v0.7.0

## 📌 一页纸总结（What / Why / Result）

### What

- Markdown/Preview：
  - 统一 `MarkdownHTMLRenderer` 能力配置，正式收口到 `CommonMark + GFM + footnotes + math`，并补上 task list、definition list、代码高亮与 safe HTML subset。
  - preview 和 PNG export 现在共用同一套 Markdown feature set、同一份本地 JS/CSS 资产和同一套安全边界。
- Export/Long Content：
  - 导出布局宽度改为对齐 preview，代码块保持“预览横滚、导出适配”，长图导出补上 top / middle / bottom 连续性护栏。
  - PDF 导出分支新增真实 page-box raster 预算预检，避免 viewport 预算看似安全、实际 PDF raster 超限时才发生裁切或失败。
- Safety/Test：
  - 新增 safe HTML subset、preview navigation policy 与 rich markdown fixture 回归。
  - 补齐长图 seam、顶部缺失、超宽代码块导出、标准 Markdown case 等 UI regression。

### Why

- 之前的 Markdown 链路能力分散：preview / export 共用总体方向，但 feature flag、样式、运行时和长图导出策略仍可能分叉，导致“预览可读、导出不全”或“预览正常、导出字体更大导致换行更多”这类问题。
- 用户明确把“完整支持”锁定为 `CommonMark + GFM + footnotes + math`，并要求不换栈，继续沿用 `markdown-it + KaTeX + WKWebView`。因此本轮目标是把能力补齐、把安全边界收口、再把 preview/export 一致性做实。
- 超长 PNG 导出的问题并不在 Markdown 解析，而在 export 专用路径：高度测量、tile 拼接、PDF raster 预算和 export-only 宽内容适配叠加后，容易造成顶部丢失、中段白缝或尾部裁切。

### Result

- Scopy 现在把 Markdown 的正式支持面提升到以 `CommonMark + GFM + footnotes + math` 为核心，并把 task list、definition list、代码高亮和 safe HTML subset 纳入产品级能力。
- 预览可读性明显更稳定：浅色高对比 surface、统一的代码块主题和更明确的 overflow 处理，减少暗色宿主背景对白底正文的干扰。
- 长图导出链路更接近 preview：只有表格和超宽代码块在 export 阶段做必要适配，同时为超长内容增加了连续性和 PDF 预算护栏。

---

## 🏗️ 实现路线

1. 抽出 `MarkdownRenderFeatureSet`，把 Markdown runtime 选项、插件和资源头标签收口到一个地方，确保 preview/export 不再各自拼接能力。
2. 在 renderer 内引入 safe HTML subset、footnotes、definition lists、highlight.js 和 task-list runtime，把两层 Markdown 支持落到同一条本地资源链。
3. 调整 preview/export 共享样式基线：固定浅色内容面、统一代码字体与表格/脚注/数学溢出检测，避免“预览看不清、导出又是另一套排版”。
4. 强化导出链路：对齐 preview 布局宽度、为 overwide code 增加 export-only wrap、为 tall export 增加高度 reconciliation、tile 区间精确映射和 PDF page-box 预算预检。
5. 补充 unit + UI regression，把 rich markdown fixture、safe navigation policy、标准 case、wide code 和 long markdown continuity 一次锁住。

## 🔮 后续

- 如果后续要继续逼近“导出图与 preview 几乎完全一致”，优先方向不是再换 Markdown 栈，而是继续针对极端长文档的导出高度演化、tile 诊断与高分辨率策略做细化。
- 如果未来要扩展 raw HTML、Mermaid、admonitions 或外部资源加载，应作为单独安全评审主题处理，不应直接落在当前 `html: false` 的受控链路上。
