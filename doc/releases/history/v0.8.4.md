# v0.8.4

## Summary

This patch finishes the ChatGPT-aligned Markdown preview scaling model and smooths the hover-preview scale interaction. Preview and export now keep the same fixed render surface while the active ChatGPT layout scale controls font metrics and line breaks, including continuous 80%-200% adjustment with light 5% snapping.

The release also carries the final WACZ parity fixes for source citation pills, ATX heading normalization, table pipe handling, wide-table behavior, code colors, inline-code contrast, and the Markdown style contract documentation.

## Key Changes

### Markdown Preview Scaling

- Keeps the Markdown hover-preview shell width tied to Scopy's shared image/file preview frame while decoupling that frame from Markdown content layout.
- Supports a continuous 80%-200% ChatGPT layout-scale slider in hover preview and Settings, with light snapping near every 5% stop and integer-percent renderer support.
- Keeps preview/export output pixels stable across layout scales; only font metrics and line breaks change.
- Preserves the previous rendered Markdown surface while the newly selected scale is debounced and rerendered, avoiding blank-frame flashes during slider drag.
- Compacts the hover-preview scale control into a lighter hover-expanded pill so the slider is available during interaction without permanently covering the reading area.
- Makes hover-preview PNG export use the same layout profile as the visible preview.

### Markdown Rendering Parity

- Re-anchors heading, paragraph, list, and blockquote rhythm to the captured ChatGPT `markdown-new-styling` DOM and root `wrap-break-word` behavior.
- Preserves heading-contained inline code as a ChatGPT inline-code pill with relative sizing and normal wrapping.
- Promotes explicit source-reference links such as `([AP News][1])` to ChatGPT-style source citation pills, including grouped `+N` source counts, while preserving ordinary parenthesized links.
- Normalizes malformed ATX headings such as `#标题` before both renderer paths parse Markdown.
- Normalizes unescaped pipe characters inside one- or two-backtick table-row code spans so code examples stay in their original table cell.
- Restores standard Markdown tables to the natural `width: 100%` path while reserving WACZ table-container breakout sizing for tables classified as wide.
- Keeps wide-table scroll anchored at the normal message-column inset and applies PNG table fitting as a transform on the same laid-out table surface.
- Aligns preview and export `hljs` token colors and inline-code contrast with the captured WACZ color contract.

## Documentation

- Updates `doc/current/markdown-chatgpt-wacz-style-contract.md` with the source-derived width, layout-scale, table, citation, and typography contract.
- Updates `doc/current/development-guide.md` and `doc/current/product-spec.md` so future work preserves the separation between preview frame width, output surface size, and Markdown layout scale.

## Performance Evidence

No dedicated release profile was added for `v0.8.4`; `profile_doc` is `null` in release metadata. The change is scoped to Markdown rendering, hover-preview interaction, and documentation. It does not alter search, storage, thumbnail generation, pagination, or scroll hot paths.
