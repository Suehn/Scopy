# v0.7.3

## Summary

- Tall Markdown/HTML PNG export now bypasses WebKit/Quartz multi-page PDF rasterization on the automatic path, avoiding the large blank upper region seen in long-note exports.
- Markdown math and safe-HTML restoration now use salted per-document placeholders and literal replacement so user content cannot collide with internal renderer tokens.
- Markdown preview now allows same-document fragment links for footnotes and anchors while continuing to block external, targetless, and cross-document navigation.
- Release CI and publishing helpers are hardened around cask sync, existing release tags, hosted runner module dependencies, and strict-concurrency timing flakes.

## Key Changes

- Lowered the automatic PDF export height cutoff to single-page captures and route taller automatic exports through the tiled snapshot path.
- Reworked math and safe-HTML placeholder generation/restoration to avoid literal placeholder collisions and keep multi-block `<details>` fallback output stable.
- Centralized same-document fragment navigation checks in the Markdown preview WebKit navigation policy.
- Split hosted CI gates, made the test module dependency explicit, and stabilized release/strict-concurrency test timing with real test signals.
- Kept release metadata tag-driven: this release remains sourced from `doc/meta/release-current.yml` and will be published by the `v0.7.3` git tag.
