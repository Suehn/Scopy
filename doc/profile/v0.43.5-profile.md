# v0.43.5 性能对比（v0.43.4 → v0.43.5）

**日期**: 2025-12-14

## 对比版本

- v0.43.4 → v0.43.5

## 测试环境

- 设备：MacBook Air（Apple M3, 24 GB）
- 系统：macOS 15.7.2（24G325）
- 架构：arm64
- 配置：Debug
- 命令：`make test-perf`（`RUN_PERF_TESTS=1`）
- 电源：Low Power Mode enabled（`pmset -g` 显示 `lowpowermode 1`）

> 注意：v0.43.4 的对比基线记录在 Low Power Mode disabled 环境下；与本次采样电源策略不同，表格仅供参考，不建议将差异解读为代码带来的确定性提升/回归。

## 性能对比表格

| 指标 | v0.43.4 | v0.43.5 |
|------|--------:|--------:|
| Fuzzy 5k items P95 | 4.82ms（Low Power Mode disabled） | 8.26ms（Low Power Mode enabled） |
| Fuzzy 10k items P95 | 44.61ms（Samples: 50；Low Power Mode disabled） | 75.82ms（Samples: 50；Low Power Mode enabled） |
| Disk 25k fuzzy P95 | 70.83ms（Samples: 50；Low Power Mode disabled） | 99.52ms（Samples: 50；Low Power Mode enabled） |
| Bulk insert 1000 items | 51.87ms（≈19,277 items/s；Low Power Mode disabled） | 84.10ms（≈11,891 items/s；Low Power Mode enabled） |
| Fetch recent (50 items) avg | 0.07ms | 0.11ms |
| Regex 20k items P95 | 2.99ms | 5.32ms |
| Mixed content disk search（single run） | 4.25ms | 7.46ms |

## 测试用例变化

- 无（本版本主要为图片 hover 预览路径优化；perf 用例集合未变）。

## 性能回归说明

- 本版本未改动搜索/存储热路径的算法结构；指标变化更可能来自电源策略差异与系统抖动。
- hover 预览体感提升不直接体现在 `make test-perf` 指标中（属于 UI 交互路径）。

## 总结

- v0.43.5 的重点是图片 hover 预览提速（缩略图占位 + JPEG downsample），性能数据作为参考记录；若需要严格对比请在相同电源策略下复测。
