# 优化路线图（功能稳定、用户无感）

## 1. 优先级框架
排序规则：用户体感收益 > 功能稳定性风险 > 实现复杂度 > 可回滚性。
分层推进：先低风险高收益，再中风险结构性优化。

## 2. Phase 1（低风险，高收益，建议先做）
1. HistoryViewModel 行级索引化与写回短路
- 目标：降低 O(n) 查找/替换频度。
- 预期收益：snapshot debug 短查询端到端再降 10-20ms（场景相关）。
- 风险：索引一致性；可通过契约测试覆盖。

2. Scroll resolver/attach 缓存
- 目标：减少重复递归查找。
- 预期收益：滚动稳定性提升，tail latency 改善。
- 风险：缓存失效时机；通过 hierarchy 变化钩子控制。

3. 预览任务预算统一（不改语义）
- 目标：减少滚动期间 task 抖动。
- 预期收益：主线程平滑度提升。
- 风险：任务取消边界；通过 UI 交互回归覆盖。

## 3. Phase 2（中风险，中高收益）
1. cleanup 计划合并（count + external）
- 目标：减少重复清理阶段与重复 I/O。
- 预期收益：heavy cleanup 从 ~1180ms 继续压缩。
- 风险：语义偏差；必须先做 shadow compare。

2. external size 增量维护
- 目标：减少目录全扫描。
- 预期收益：大库清理与统计路径稳定提速。
- 风险：统计一致性；需要不变量校验与恢复策略。

## 4. Phase 3（中高风险，按需推进）
1. 搜索候选裁剪与 top-k 排序策略进一步优化。
2. 清理批次/并发在不同硬件上的自适应策略。
3. 更大规模库（>100k）专项基准。

## 5. 里程碑与门禁
- M1：Phase 1 完成并门禁全绿。
- M2：Phase 2 完成并验证“语义不变 + 性能提升”。
- M3：按产品需求决定是否进入 Phase 3。
