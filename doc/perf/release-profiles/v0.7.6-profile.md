# v0.7.6 Row Descriptor And Thumbnail Scheduler Profile

## Scope

This profile documents the frontend architecture and profile reliability work shipped in `v0.7.6`.

The release does not intentionally change user-facing clipboard, search, preview, export, selection, or settings behavior. It moves row-ready presentation data into `HistoryItemRowDescriptor`, routes visible row thumbnail sequencing through `HistoryRowThumbnailLifecycleScheduler`, and isolates Scopy app processes inside the frontend profile runner.

## Evidence Inputs

- Snapshot DB: `perf-db/clipboard.db` (6421 items / 148647936 bytes)
- Frontend smoke profile: `logs/perf-frontend-profile-2026-05-07_03-18-32/frontend-scroll-profile-summary.md`
- Frontend standard profile: `logs/perf-frontend-profile-2026-05-07_03-22-16/frontend-scroll-profile-summary.md`

The frontend profile `baseline` and `current` variants are feature-flag variants inside the same build. Read this profile as a regression guard for the row and thumbnail paths, not as a clean git-before/git-after A/B for every UI metric.

## Frontend Standard Profile

The row descriptor kept display-model work effectively flat and sub-millisecond across all three real-snapshot scenarios. The thumbnail lifecycle path improved `image.thumbnail_load_total_ms.p95` in the same run, with the largest improvement in the text-biased scenario.

## Profile Runner Reliability

`scripts/perf-frontend-profile.sh` now isolates Scopy processes at three points:

- before the first `xcodebuild` run;
- between each baseline/current variant;
- on script exit through the cleanup trap.

If a frontend profile fails before generating `frontend-scroll-profile-summary.md`, the run is not evidence. First clear residual Scopy/XCTest processes and rerun the profile.
