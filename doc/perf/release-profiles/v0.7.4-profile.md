# v0.7.4 Frontend Scroll Profile

## Scope

This profile documents the frontend scroll attribution and low-risk scroll-time optimizations shipped in `v0.7.4`.

The release does not change user-facing clipboard, preview, export, selection, or settings behavior. The changes focus on measuring the real scroll bottleneck and moving safe presentation work out of the hot row render path.

- Snapshot DB: `perf-db/clipboard.db`（6421 items / 148647936 bytes）

## Attribution Reading

Long-frame attribution shows app-level row/render buckets explain only a small share of long-frame time, while main-thread coverage remains high:

| Scenario | Long frames | App coverage | Main-thread coverage | Top correlated app buckets |
|---|---:|---:|---:|---|
| real-snapshot-accessibility | 247 | 1.6% | 95.4% | display model, markdown detect, metadata, row body |
| real-snapshot-mixed | 232 | 2.3% | 94.9% | display model, markdown detect, metadata, row body |
| real-snapshot-text-bias | 171 | 2.7% | 96.5% | display model, markdown detect, metadata, row body |

The practical conclusion is that row body work, row equality, accessibility query cost, and most thumbnail decode work are now measured and bounded. Remaining frame instability should be investigated as main-thread/system RunLoop work or SwiftUI layout/diff/render work that the current app-level buckets do not fully cover.
